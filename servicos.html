Gestão da Qualidade · Etapa 2

Testes, Falhas e Correções

Aplicamos um ciclo completo de qualidade em três níveis para cada sistema entregue — garantindo excelência e confiabilidade nos sistemas públicos.

01
Teste Unitário

Validação de funções isoladas

Cada função crítica é testada individualmente antes de qualquer integração. Erros de lógica são capturados na origem, antes de gerar impacto financeiro ou operacional.

# Falha identificada return valor_mensal + meses # AssertionError: 1012 != 12000 # Correção aplicada def calcular_valor_total(self, v, m): if v < 0 or m < 0: raise ValueError("Valores negativos") return v * m # ✓ PASS
02
Teste de Integração

Comunicação entre camadas

Verificamos a compatibilidade de interfaces entre componentes. Divergências de nomenclatura ou contratos de API são detectadas antes de chegarem ao ambiente de produção.

# Falha: método inexistente AttributeError: 'MockDB' has no attr 'get_by_id' # Correção: sincronização de interface def buscar_contrato(self, id): return self.db.find_id(id) # ✓ PASS
03
Teste de Aceitação

Validação pelo usuário final

Garantimos que a interface e a experiência atendem às expectativas reais dos usuários — com formatações corretas, rótulos descritivos e tratamento de erros visível.

# Falha: saída sem formatação "Resultado: 1012" # Correção: UI humanizada def exibir_resumo(self, v, m): total = self.service.calcular(v, m) return f"Valor Total: R$ {total:.2f}" # ✓ "Valor Total: R$ 12.000,00"

Documentação das falhas e correções

01
Unitário

✗ Falha identificada

A função calcular_valor_total usava adição (+) em vez de multiplicação (*), subestimando drasticamente os valores de contrato. Resultado: 1012 em vez de 12000.

✓ Correção aplicada

Operador corrigido para multiplicação. Adicionada validação de entradas negativas com ValueError. Teste test_calcular_valor_total aprovado com 100% de cobertura.

02
Integração

✗ Falha identificada

ContratoService chamava get_by_id, mas MockDB implementava find_id. Bloqueio total de consultas ao banco — sistema inoperante para buscas.

✓ Correção aplicada

Método ajustado para find_id em toda a camada de serviço. Interfaces sincronizadas entre componentes. Teste test_integracao_banco aprovado.

03
Aceitação

✗ Falha identificada

Interface exibia valor bruto sem formatação de moeda nem rótulo descritivo. Output "Resultado: 1012" comprometia a usabilidade e legibilidade para o usuário final.

✓ Correção aplicada

Formatação BRL implementada (R$ {total:.2f}), rótulo semântico adicionado e tratamento try/except incluído. Teste test_aceitacao_usuario aprovado.

100% Taxa de sucesso nos testes
ZERO Falhas críticas em produção
